# qlizmet

Quizlet-подобное десктопное приложение для заучивания карточек. Курсовая работа.

Все шесть режимов оригинала, интервальные повторения по алгоритму SM-2 (как в
Anki), локальное хранение на одного пользователя и карточки со смешанным
содержимым — текстом, изображениями и LaTeX-формулами.

## Возможности

**Режимы обучения**

| Режим | Что делает |
|-------|-----------|
| Карточки | листаешь колоду, переворачиваешь, отмечаешь «знаю / не знаю» |
| Заучивание | адаптивно ведёт карточку к «выучено»: выбор варианта → ввод ответа |
| Письмо | ввод ответа с проверкой, допуском опечаток и кнопкой «Я был прав» |
| Тест | билет из вопросов трёх типов, проверка и разбор ошибок в конце |
| Подбор пар | сопоставление лица и оборота на время |
| Гравитация | падающие термины: очки, жизни, растущая скорость |

Режимы, недоступные для набора, гаснут с объяснением причины: например, «Письмо»
требует хотя бы одного текстового ответа — формулу не напечатаешь.

**Прочее:** интервальные повторения SM-2 с историей ответов, статистика по набору
(новые / в работе / закреплённые, что пора повторить, доля верных ответов),
импорт и экспорт наборов в TSV, светлая и тёмная темы с переключателем.

## Стек

- Python 3.11+
- [PySide6](https://doc.qt.io/qtforpython/) (Qt) — интерфейс
- SQLite (`sqlite3` из стандартной библиотеки) — хранение
- matplotlib (mathtext) — рендер формул
- pytest — тесты

## Установка и запуск

```bash
python -m venv .venv
# Windows: .\.venv\Scripts\Activate.ps1
# Linux/macOS: source .venv/bin/activate

pip install -e ".[dev]"
python -m qlizmet
```

Установка через `-e` обязательна: пакет лежит в `src/`, и без неё
`python -m qlizmet` его не найдёт.

## Тесты

```bash
pytest
```

Тесты интерфейса выполняются headless (`QT_QPA_PLATFORM=offscreen`), поэтому
проходят и в CI. Реальное время в играх не ожидается: экраны выносят шаг времени
в публичный `tick()`, а тесты шагают по нему вручную.

## Разметка карточек

Стороны карточки набираются обычным текстом с лёгкой разметкой:

```
обычный текст                 → текст
производная $\sin x$ равна    → текст + формула
![подпись](картинка.png)      → изображение
```

Предпросмотр обновляется прямо в редакторе карточки.

## Где лежат данные

База и медиафайлы хранятся в пользовательской папке приложения, а не рядом с кодом:

| ОС | Путь |
|----|------|
| Windows | `%APPDATA%\qlizmet` |
| macOS | `~/Library/Application Support/qlizmet` |
| Linux | `$XDG_DATA_HOME/qlizmet` (по умолчанию `~/.local/share/qlizmet`) |

Переменная окружения `QLIZMET_HOME` перекрывает путь — этим пользуются тесты,
чтобы не трогать реальные данные. Изображения карточек кладутся в подпапку
`media/` и подключаются по имени файла.

## Архитектура

Слоистая, с односторонним правилом зависимостей `ui → app → core`. Ядро (`core/`) —
чистый Python без единого импорта Qt, поэтому вся предметная логика (режимы, SM-2,
оценка ответов, статистика) тестируется без запуска окна. Хранилище подключается
к ядру через интерфейсы (`storage/repository.py`), поэтому БД остаётся сменной
деталью.

```
src/qlizmet/
  core/     модели, разметка, режимы, SRS, оценка ответов, статистика
  storage/  репозитории и их реализации, импорт/экспорт
  app/      сервисы и use-cases
  ui/       Qt: единственный слой, импортирующий PySide6
```

Реал-тайм (секундомер «Подбора пар», падение в «Гравитации») живёт в слое `ui/`:
ядро игр остаётся чистым конечным автоматом, а `QTimer` лишь вызывает его шаг.

Оформление вынесено целиком в `ui/theme.py`: экраны не содержат ни одного цвета,
они лишь расставляют `objectName`, а вид задаёт одна таблица QSS. Динамические
состояния (верный ответ, выбранная плитка) выражаются свойствами Qt. Отдельный
тест следит, чтобы в экранах не появился `setStyleSheet`.
