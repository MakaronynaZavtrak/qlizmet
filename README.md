# qlizmet

Quizlet-подобное десктопное приложение для заучивания карточек.

Все шесть режимов оригинала, интервальные повторения по алгоритму SM-2 (как в
Anki), локальное хранение на одного пользователя и карточки со смешанным
содержимым — текстом, изображениями и LaTeX-формулами.

## Возможности

**Режимы обучения**

| Режим | Что делает |
|-------|-----------|
| Карточки | листаешь колоду, переворачиваешь, отмечаешь «знаю / не знаю» |
| Заучивание | адаптивно ведёт карточку к «выучено»: выбор варианта → ввод ответа |
| Письмо | ввод ответа с проверкой, допуском опечаток и кнопкой «Я был прав» |
| Тест | билет из вопросов трёх типов, проверка и разбор ошибок в конце |
| Подбор пар | сопоставление лица и оборота на время |
| Гравитация | падающие термины: очки, жизни, растущая скорость |

Режимы, недоступные для набора, гаснут с объяснением причины: например, «Письмо»
требует хотя бы одного текстового ответа — формулу не напечатаешь.

**Прочее:** интервальные повторения SM-2 с историей ответов, статистика по набору
(новые / в работе / закреплённые, что пора повторить, доля верных ответов),
импорт и экспорт наборов в TSV.

## Стек

- Python 3.11+
- [PySide6](https://doc.qt.io/qtforpython/) (Qt) — интерфейс
- SQLite (`sqlite3` из стандартной библиотеки) — хранение
- matplotlib (mathtext) — рендер формул
- pytest — тесты

## Установка и запуск

```bash
python -m venv .venv
# Windows: .\.venv\Scripts\Activate.ps1
# Linux/macOS: source .venv/bin/activate

pip install -e ".[dev]"
python -m qlizmet
```

Установка через `-e` обязательна: пакет лежит в `src/`, и без неё
`python -m qlizmet` его не найдёт.

## Тесты

```bash
pytest
```

Тесты интерфейса выполняются headless (`QT_QPA_PLATFORM=offscreen`), поэтому
проходят и в CI. Реальное время в играх не ожидается: экраны выносят шаг времени
в публичный `tick()`, а тесты шагают по нему вручную.
